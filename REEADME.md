# cpp-poker-game-server
> **基於 C++ (BCB) 與 TCP Socket 實作之多人在線連線撲克牌遊戲伺服器**

本專案使用 **C++ 語言** 結合 **C++ Builder (BCB)** 開發，拋棄現成高階遊戲引擎，直接從底層 TCP 網路協定（Socket）架構實現 4 人連線房間、自訂資料封包、即時牌局同步管理與動態例外斷線處理機制。

---

## 🎯 射龍門遊戲規則簡介 (Game Rules)
「射龍門」是一款結合機率評估與心理戰的經典撲克牌遊戲，基礎運作邏輯如下：
1. **底池初始化**：每局開始前，所有玩家共同投入基本底注至中央彩金池（`totalmoney`）。
2. **發放雙柱（邊界牌）**：系統為當前回合玩家隨機發出兩張牌（球門的左、右柱）並全場公開。
3. **下注與射門**：玩家依據兩牌間的範圍寬度評估勝率並輸入下注金額。接著系統發出關鍵第三張牌：
   * **射門成功**：第三張牌數字介於前兩牌之間，玩家贏得同等下注金額。
   * **射門失敗**：第三張牌在範圍之外，下注金額直接沒收充公。
   * **撞柱（最刺激惩罰）**：第三張牌與前兩牌任一張數字相同，玩家判定慘輸，**必須賠給彩金池「雙倍」的下注金額**。

## 💻 程式運行流程與核心實作 (Gameplay Flow & Architecture)

本專案核心程式碼（`Unit1.cpp` / `Unit2.cpp`）完全不依賴套件，從零手寫底層通訊。整體系統的運行流程與技術實作如下：

### 1. 房間建立與多端長連線初始化
伺服器端開啟 `TServerSocket` 監聽，用戶端透過輸入 IP 位址連線。系統採用 TCP/IP 長連線架構，當 4 位玩家成功連入房間後，伺服器會初始化牌池，利用**二維映射洗牌演算法**結合 `TStringList` 機制進行牌池去重驗證，從底層百分之百杜絕重複發牌的漏洞。

### 2. 事件驅動下的牌局流程控制
遊戲開始後，伺服器透過自訂的**字串分割通訊協定**進行多端毫秒級廣播。系統利用物件導向建構嚴密的遊戲狀態機（State Machine），依序控制「準備 ➔ 發邊界牌 ➔ 等待操作 ➔ 判定結算」的狀態鏈。每當牌局狀態改變，全場玩家的 UI 畫面、Log 紀錄與彩金池金額皆會異步即時同步。

### 3. 逾時自動操作與防死鎖機制 (關鍵亮點 💡)
在等待玩家決策（下注、抽牌、棄牌）的過程中，為防止個別玩家因網路延遲、掛機或惡意不操作而導致整局遊戲發生死鎖（Deadlock），系統實作了**回合秒數倒數機制**。當計時器（`TTimer`）偵測到玩家操作逾時，伺服器端會強制觸發預設防呆動作（如：自動判定為最低下注或棄牌），將牌局安全推移至下一階段，徹底杜絕遊戲卡死。

### 4. 背景例外偵測與斷線重置
在遊戲進行的任一時刻，伺服器的背景監聽鏈與隨機心跳檢查機制皆在持續運作。一旦偵測到玩家網路波動或異常關閉視窗，系統會立刻主動關閉錯誤的 Socket 插槽防止記憶體殘留，同時修正連線計數器、自動跳過斷線玩家，大幅提升多人遊戲的穩定性與伺服器防崩潰能力。
