# cpp-poker-game-server
> 基於 C++ 與 TCP Socket 實作之多人在線連線撲克牌遊戲伺服器

使用 C++ 語言結合 C++ Builder (BCB) 開發多人在線連線撲克牌遊戲系統，採用 Client-Server 架構實現即時網路通訊與遊戲狀態同步。系統支援多名玩家同時在線進行押注、抽牌及勝負判定，並透過封包同步機制確保遊戲資料一致性。此外，實作玩家異常斷線偵測與自動處理機制，提升多人連線環境下的穩定性與使用體驗。

## 射龍門遊戲規則簡介
「射龍門」是一款結合機率評估與心理戰的經典撲克牌遊戲，基礎遊戲流程如下：
1. **發放雙柱（邊界牌）**：系統為當前回合玩家隨機發出兩張牌（球門的左、右柱）並全場公開。
2. **底池初始化**：每局開始前，所有玩家共同投入基本底注至中央彩金池。
3. **下注與射門**：玩家依據兩牌間的範圍寬度評估勝率並輸入下注金額。接著系統發出關鍵第三張牌：
   * **射門成功**：第三張牌數字介於前兩牌之間，玩家贏得同等下注金額。
   * **射門失敗**：第三張牌在範圍之外，下注金額直接沒收充公。

## 程式運行流程與核心實作

### 1. 房間建立
伺服器端開啟 `TServerSocket` 監聽，用戶端透過輸入 IP 位址連線。系統採用 TCP/IP 長連線架構，當 4 位玩家成功連入房間後，伺服器會初始化牌池。

### 2. 多玩家狀態機與發牌邏輯
* **洗牌機制**：花色與數字利用二維陣列演算法進行發牌。
* **去重演算法**：發牌時利用 `TStringList *dealtCards` 儲存已發出之撲克牌，每次抽牌進行雜湊比對，確保多人在線局中的公平性與正確性。

### 3. 事件驅動下的牌局流程控制
遊戲開始後，伺服器透過自訂的字串分割通訊協定進行廣播。系統利用物件導向建構嚴密的遊戲狀態機，依序控制「準備 ➔ 發邊界牌 ➔ 等待操作 ➔ 判定結算」的狀態鏈。每當牌局狀態改變，全場玩家的畫面、Log 紀錄與彩金池金額皆會異步即時同步。

### 4. 逾時自動操作與防死鎖機制
在等待玩家決策（下注、抽牌、棄牌）的過程中，為防止個別玩家因網路延遲、掛機或惡意不操作而導致整局遊戲發生死鎖，有做了回合秒數倒數機制。當偵測到玩家操作逾時，伺服器端會強制觸發預設防呆動作（如：自動判定為最低下注或棄牌），將牌局安全推移至下一階段，徹底杜絕遊戲卡死。

### . 背景例外偵測與斷線重置
在遊戲進行的任一時刻，伺服器的背景監聽鏈與隨機心跳檢查機制皆在持續運作。一旦偵測到玩家網路波動或異常關閉視窗，系統會立刻主動關閉錯誤的 Socket 插槽防止記憶體殘留，同時修正連線計數器、自動跳過斷線玩家，大幅提升多人遊戲的穩定性與伺服器防崩潰能力。
